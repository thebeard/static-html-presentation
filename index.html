Session 5

We watch a real-time presentation in which a web page is created from stratch in static HTML. PHP is available if we want to try insert some dynamic content and we'll try to move unto CSS and a small snippet of Javascript.

This will be done at a fairly quick pace to illustrate as much of the process as possible. Questions are however very much encouraged!

Session 4

We install markdown viewer extension in our browser, continue our collaborative session and wrap up the basics of markdown syntax; preparing us for learning it's sibling language where we'll dive deeper into semantic content structure for the web: HTML!

Session 3

We backtrack a bit and review what we've discussed in the first few sessions. We also join a collaborative session and start working on plain text and markdown files. Markdown files are rendered as plain text files in a browser as it's not natively integrated.

Session 2

The web and all it's prinicples forms a large ecosystem of roles, responsibilities, skill sets and interests. To be relevant / employable it's good to involve yourself and understand as much as possible, while focusing on your unique skills.

Session 1

We jump in, looking at what an HTML file looks like, make some changes in an IDE, start up a web server, push changes to a public facing website and see our changes reflected. Possibly a shock to the system - something we'll do every few sessions.
